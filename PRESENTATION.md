План защиты проекта SciTriveo: Intellectual Research Copilot

1. Обзор проекта и используемых данных

Цель проекта: разработка высокопроизводительной системы поиска и анализа научных публикаций с поддержкой русского и английского языков.
Корпус данных: уникальный набор научных статей, включающий метаданные (DOI, авторы, журналы, год издания, цитируемость) и текстовые аннотации (Abstracts).
Технологический стек: Python (backend), Flask (web framework), Scikit-learn, Gensim, Transformers, SymSpell.

2. Архитектурные решения и чистота кода 

Структура проекта: использование паттерна Composition Root. Класс AppContainer (src/app/container.py) инкапсулирует все сервисы и индексы, обеспечивая единую точку доступа к логике.
Модульность: четкое разделение на слои индексации, поиска, лингвистической обработки и веб-интерфейса.
Стандарты: код соответствует PEP8, снабжен комментариями на английском языке и детальными Docstrings в формате reStructuredText.
Документация: сгенерированная техническая документация в формате HTML доступна в папке /docs.

3. Демонстрация базового функционала поиска в CLI 

Для демонстрации используется интерактивный режим, исключающий повторную загрузку моделей при каждом запросе.
Запуск:
```bash
python -m src.cli interactive
```

Демонстрация типов индексации:
- Лексический поиск (BM25):
```
search "neural networks" --index bm25 --top-k 3
```
- Семантический поиск (FastText):
```
search "morphology of languages" --index fasttext --top-k 3
```
- Статическая модель (Word2Vec):
```
search "quantum computing" --index word2vec --top-k 3
```
В выводе демонстрируется время поиска (latency) и полные метаданные найденных объектов.

4. Веб-интерфейс на Flask и работа с коллекциями
Запуск веб-приложения:

```bash
python -m src.web.app
```

Маршруты демонстрации (http://127.0.0.1:5000):
- Главная страница: описание проекта и переход к поисковому интерфейсу.
- Страница поиска: гибкая настройка параметров поиска и применение фильтров по метаданным (год, журнал, количество цитирований).
- Результаты: отображение текстов, оценок близости (score) и времени поиска.
- Функционал коллекций: создание личных коллекций и добавление в них отобранных статей.

5. Реализация требований на оценку 10: RecSys и Speller
- Исправление опечаток
- Рекомендательная система:

Демонстрация в CLI или Web. Система анализирует "центроид" коллекции пользователя и предлагает статьи через алгоритм Reciprocal Rank Fusion (RRF).
Команда в CLI:
```
recommend --id [UUID_COLLECTION] --top-k 5
```

6. Реализация требований на оценку 10: Интеграция RAG (1 минута)
Использование SOTA архитектуры Apple CLaRa-7B-Instruct для синтеза знаний.
Процесс: система извлекает все аннотации из выбранной коллекции и подает их в модель для ответа на сложный вопрос.

Команда в CLI:
```
ask --id [UUID_COLLECTION] "What are the common research methods used in these papers?"
```
